# HappyBot — แชทบอทตอบแชทในเว็บ + แดชบอร์ดแอดมิน

ระบบตัวอย่างที่ใช้งานได้จริง สร้างตามภาพหน้าจอที่ต้องการ ประกอบด้วย
หน้าต่างแชทฝั่งลูกค้า, ระบบตอบกลับอัตโนมัติ (บอท), การส่งต่อให้คนจริง (human handoff)
และแดชบอร์ดให้แอดมินดูแลทุกห้องสนทนาแบบเรียลไทม์

**จุดเด่น:** ไม่ต้องติดตั้ง dependency ใดๆ — ขอแค่มี **Node.js เวอร์ชัน 18 ขึ้นไป** ก็รันได้ทันที

---

## วิธีรัน (เริ่มใน 30 วินาที)

```bash
cd happybot
node server.js
```

จากนั้นเปิดเบราว์เซอร์ 2 แท็บ:

- **แดชบอร์ดแอดมิน:** http://localhost:3000/admin.html
- **แชทลูกค้า:** http://localhost:3000/widget.html

ลองพิมพ์ในหน้าลูกค้า เช่น "ค่าส่งเท่าไหร่" แล้วดูบอทตอบ จากนั้นสลับไปดูที่หน้าแอดมิน
จะเห็นข้อความเด้งขึ้นแบบเรียลไทม์ กดปุ่มสลับโหมดเพื่อตอบเองได้

---

## โครงสร้างโปรเจกต์

```
happybot/
├── server.js          # เซิร์ฟเวอร์หลัก: REST API + เรียลไทม์ (SSE)
├── bot.js             # สมองของบอท: กฎ + จุดเสียบ AI + ส่งต่อให้คน
├── db.js              # ชั้นเก็บข้อมูล (ไฟล์ JSON)
├── line.js            # ตัวเชื่อม LINE Official (ขั้นตอนที่ 6)
├── public/
│   ├── widget.html    # หน้าต่างแชทฝั่งลูกค้า (ฝังในเว็บได้)
│   └── admin.html     # แดชบอร์ดแอดมิน (ตามภาพ)
└── data/store.json    # ข้อมูล (สร้างอัตโนมัติ)
```

## ทั้ง 7 ขั้นตอน — ทำอะไรไปแล้วบ้าง และต้องทำต่ออะไร

### ✅ 1. หน้าต่างแชทบนเว็บ (chat widget)
อยู่ที่ `public/widget.html` เป็นกล่องแชทที่รับ-ส่งข้อความได้ และจำห้องสนทนาเดิมไว้ในเครื่อง
ผู้ใช้ (ผ่าน `localStorage`) เพื่อให้คุยต่อได้เมื่อกลับมา

**ฝังในเว็บจริง:** วางโค้ดนี้ในหน้าเว็บของคุณ แล้วชี้ src ไปที่เซิร์ฟเวอร์
```html
<iframe src="https://your-server.com/widget.html"
        style="position:fixed;bottom:20px;right:20px;width:400px;height:640px;border:none;border-radius:20px;box-shadow:0 20px 50px rgba(0,0,0,.2)"></iframe>
```

### ✅ 2. ระบบเรียลไทม์
ใช้ **SSE (Server-Sent Events)** — เซิร์ฟเวอร์ push ข้อความหาเบราว์เซอร์ทันที
โดยไม่ต้อง refresh และไม่ต้องลง library (เบราว์เซอร์มี `EventSource` ในตัว)
ดูได้ใน `server.js` ที่ endpoint `/api/stream` (ฝั่งลูกค้า) และ `/api/admin-stream` (ฝั่งแอดมิน)

> หมายเหตุ: ถ้าต้องการสองทาง/ความหน่วงต่ำมาก เช่นมีฟีเจอร์ "กำลังพิมพ์" แบบสด
> ค่อยอัปเกรดเป็น WebSocket (เช่นไลบรารี `ws` หรือ `socket.io`) ภายหลังได้

### ✅ 3. ฐานข้อมูล
ตอนนี้ใช้ไฟล์ JSON (`db.js`) เพื่อให้รันได้ทันที เก็บบทสนทนา + ข้อความครบ
**เมื่อขึ้นใช้งานจริง** ควรเปลี่ยนเป็นฐานข้อมูลจริง โดยแก้แค่ฟังก์ชันใน `db.js`:
- **PostgreSQL** — เหมาะกับข้อมูลมีโครงสร้าง (ลูกค้า/คำสั่งซื้อ/ข้อความ)
- **MongoDB** — ยืดหยุ่นกับข้อความหลายรูปแบบ
- **SQLite** — เบา เหมาะกับร้านเล็ก (ไลบรารี `better-sqlite3`)

### ✅ 4. สมองของบอท (`bot.js`)
มี 3 ระดับ ทำงานไล่กัน:
1. ถ้าลูกค้าขอ "คุยกับคนจริง" → ส่งต่อทันที
2. ถ้าตั้งค่า AI ไว้ → ใช้ AI ตอบ (ดูหัวข้อ "เปิดใช้ AI" ด้านล่าง)
3. ถ้าไม่ → จับคู่คำสำคัญตามกฎใน `RULES`
4. ถ้ายังตอบไม่ได้ → ส่งต่อให้แอดมิน (human handoff)

**ปรับคำตอบของร้านคุณ:** แก้ array `RULES` ใน `bot.js` ได้เลย

**เปิดใช้ AI (ทางเลือก):** ตั้งค่า environment variable แล้วรันใหม่
```bash
ANTHROPIC_API_KEY=คีย์ของคุณ node server.js
```
โค้ดเรียก AI อยู่ในฟังก์ชัน `aiReply()` ปรับ system prompt / โมเดลได้ตามต้องการ

### ✅ 5. แดชบอร์ดแอดมิน (`public/admin.html`)
หน้าจอตามภาพ: เมนูซ้าย, รายการแชทพร้อมตัวนับข้อความที่ยังไม่อ่าน, หน้าต่างสนทนา,
และแผงข้อมูลลูกค้า แอดมินกดสลับโหมด **บอทตอบ ⇄ ตอบเอง** ได้ทุกห้อง

### ⚙️ 6. เชื่อมช่องทางภายนอก (LINE) — โค้ดพร้อม ต้องตั้งค่าเอง
ไฟล์ `line.js` เขียนไว้พร้อมใช้ แต่ต้องมีของจริง 3 อย่าง:
1. สมัคร **LINE Official Account** แล้วเปิด **Messaging API** ที่ [LINE Developers Console](https://developers.line.biz/)
2. คัดลอก **Channel access token** และ **Channel secret**
3. มีเซิร์ฟเวอร์ที่เข้าถึงจากภายนอกได้ (public URL) — ดูขั้นตอนที่ 7

แล้วตั้งค่า:
```bash
LINE_TOKEN=xxx LINE_SECRET=yyy node server.js
```
นำ public URL + `/webhook/line` ไปใส่ช่อง **Webhook URL** ใน LINE Console
(Facebook Messenger ก็ใช้แนวคิดเดียวกัน — รับ webhook, ตรวจลายเซ็น, ตอบกลับผ่าน API ของแต่ละเจ้า)

> โค้ดส่วนนี้ต้องต่ออินเทอร์เน็ตและใช้คีย์จริง จึงทดสอบในเครื่องที่มีเน็ตของคุณเอง

### ⚙️ 7. ทดสอบและ deploy ขึ้นเซิร์ฟเวอร์
ทดสอบในเครื่อง: `node server.js` แล้วลองทุกฟีเจอร์
นำขึ้นออนไลน์ เลือกทางใดทางหนึ่ง:

- **Railway / Render** (ง่ายสุด): เชื่อม GitHub repo, ตั้ง start command เป็น `node server.js`,
  ตั้ง environment variables (ANTHROPIC_API_KEY, LINE_TOKEN, LINE_SECRET) — ได้ public URL อัตโนมัติ
- **VPS (DigitalOcean / AWS EC2)**: ติดตั้ง Node, ใช้ `pm2` รันค้างไว้, ตั้ง Nginx เป็น reverse proxy + ใบรับรอง HTTPS
- ระหว่างพัฒนาอยากต่อ LINE ชั่วคราว ใช้ `ngrok http 3000` เพื่อได้ public URL เร็วๆ ได้

**ก่อนขึ้นจริง ควรเพิ่ม:** ระบบล็อกอินแอดมิน, ฐานข้อมูลจริง (ข้อ 3), จำกัดสิทธิ์การเข้าถึง API,
และเก็บคีย์ลับไว้ใน environment variables เท่านั้น (อย่า hardcode ในโค้ด)

## เทคโนโลยีที่ใช้
- **ภาษา:** JavaScript (Node.js) ทั้ง backend และ frontend — เรียนรู้ภาษาเดียวจบ
- **เรียลไทม์:** Server-Sent Events (SSE)
- **ไม่มี dependency ภายนอก** ในเวอร์ชันนี้ เพื่อให้เริ่มต้นง่ายที่สุด
